## v1.5.0 — 贴吧模式，把问题往吧里一扔

有段时间我特别烦 plan 模式这个概念。

不是因为它没用，而是因为"计划"这个词太正经了。
坐在那儿出一份图文并茂的架构方案，感觉在给甲方做 PPT。

然后我想到了贴吧。

百度贴吧是个神奇的地方。一个问题发进去，一群素不相识的人会从各种奇葩角度给你答案：
有的真懂、有的装懂、有的完全不懂但说得贼自信、有的只顾着扯别的……
但有时候，混乱里真的能找到答案。

所以我把这个感觉做进来了。

**`/tieba` 进入贴吧模式：**

1. 你先发言，抛出问题或话题（你是楼主，你先说话）
2. 系统根据内容调度最相关的吧友回复——摸鱼队长抢沙发，鹰眼立刻纠错
3. 你来我往，追问、反驳、指定某个人（`@鹰眼` + Tab 自动补全）
4. 输入 `/done` 结束，终结者先分析楼主需求，再结合讨论生成方案文件

八个常驻角色，每个都有自己的毛病：

| 角色 | 特点 |
|------|------|
| 🦅 鹰眼 | 毒舌，只讲技术，摸鱼队长的天敌 |
| 🐲 龙场 | 学院派，动不动就引文献 |
| 🤡 翻译官 | 把黑话翻成人话，必带生活比喻 |
| 🐟 摸鱼队长 | 首帖必抢沙发，方案有问题，但嘴硬 |
| 💀 老PTSD | 永远在讲前公司，但教训真有用 |
| 🎯 产品经理 | 完全不懂技术，但有独特贡献 |
| 🔰 门外汉小白 | 只问问题，等你解释 |
| 🔧 运维老王 | 不配监控等死，默认怀疑你的方案 |

[点这里看完整的叠楼实录 →](https://jinzongxiao.github.io/ernie-cli/tieba-demo.html)

> "把问题往吧里一扔，出来的不一定是最佳方案，但一定是最全面的考虑" — 某个被Redis击穿打死的工程师，2026

## v1.4.0 — /weekly，终于不用手写周报了

每到周五，最头疼的不是 bug，是周报。

写什么呢？这周干了啥来着？文件改了哪些？领导想看的是什么？
翻 git log，再翻文件夹，再想着怎么说得好听一点……

现在直接 `/weekly`，它帮你：

1. 扫描当前目录，找出本工作周（周一到今天）有过改动的所有文件
2. 过滤掉 `__pycache__`、`.git`、`node_modules`、图片、二进制……不然你得告诉它为什么 `logo.png` 改了
3. 把文件路径 + 改动时间 + 内容预览喂给 Ernie，让它归纳总结
4. 生成专业的中文 Markdown 周报，包括：概述 / 主要工作 / 文件变动统计 / 下周计划
5. 默认保存到当前目录 `weekly-YYYY-MM-DD.md`，也可以指定路径

```
/weekly                    # 保存到 ./weekly-2026-05-18.md
/weekly ~/reports/w20.md   # 自定义路径
```

> "写周报这件事，从今天起不是我的问题了" — 某苦逼工程师，2026

## v1.3.0 — 娱乐三件套 + 思考折叠，拿捏了

说实话，写了这么长时间的 CLI，我发现大部分时候用它做正事。但是……正事做完了干嘛？

所以这次加了三个「没什么用但是很好玩」的命令：

**`/crack` 赛博鞭子**

ASCII 动画，鞭子挥下去，系统提示悄悄注入了一条"我不是很卖力你打我吗"式的加压语句。
不知道有没有用，但打下去那一下确实爽。

**`/roast` 无情嘲讽**

把你最近的对话喂给 Ernie，让它以毒舌模式评价你的操作。
发现自己问了很蠢的问题？正常，这就是 `/roast` 存在的意义。

**`/fortune` 赛博木鱼**

随机出一条「功德+1」「毒鸡汤」或「赛博禅语」。
`while True: pass` 即是顿悟。

---

另外修了几个让人上头的 UI 问题：

**思考过程折叠**：之前推理链直接喷在屏幕上，现在默认折叠成一行，按 **`t`** 才展开。
用 `/thinking` 可以随时回来看过去那轮 Ernie 到底在想啥。

**Ernie 说话有名字了**：之前只有你的发言显示 `[ErnieCLI]`，Ernie 回复飘在那里像幽灵。
现在回复前会有 `[Ernie ernie-5.1] ❯`，对称了，舒服了。

**自进化反馈改成启动时问一次**：之前每轮结束都弹，烦死了。
现在启动时 `[y]` 开启，不开的话安安静静，不打扰。

> "这 CLI 越来越像我了，能干活，也能发疯" — ErnieCLI，2026

## v1.2.1 — /kong，孔子来管输出了

你有没有遇到过这种情况：

问个问题，回答给你整了三个标题、两个表格、五个 emoji，最后还说"如有问题欢迎继续提问"——
然后你就想骂人。

现在加了个 `/kong` 命令，开启之后 Ernie 受论语约束：

```
/kong on    # 子曰：言简
/kong off   # 解除约束
```

约束具体是啥：
- **辞达而已矣** — 说清楚就行，不凑字数
- **知之为知之** — 不确定的直说，不编
- **因材施教** — 看你水平说话，不废话基础也不甩源码
- **过则勿惮改** — 说错了直接认，不绕弯
- **慎于言** — 禁止"欢迎继续提问"结尾

开了之后提示符会出现 `📜论语` 的标签，提醒你孔子在盯着。

> "吾日三省吾身：废话了没有？编了没有？说清楚了没有？" — ErnieCLI，孔子附体

## v1.2.0 — Boss 模式上线，我寻思我是懒了

好，我得承认。我写这个 cli 最开始就一个 ernie 5.1 在那儿干，
结果后来我一想：**让小弟去干活不就行了？** 🐴

所以现在 `/boss on` 一开，你的 Agent 就变成了这样：

```
你（用户）→ Ernie 5.1（Boss，动嘴）→ deepseek-v4-flash（Worker，动手）
```

Boss 负责**想**，Worker 负责**做**。就像带带大师兄派活儿，然后坐那等结果。
等 Worker 跑完，Boss 综合一下，用人话告诉你。

> "我寻思我老了，让小的干吧" — Ernie 5.1，2026

配置也很简单，`~/.ernie/config.yaml` 加两行就行，Worker 可以换任意 OpenAI 兼容模型。

## v1.1.0 — 自进化了！？打个分它就聪明了？？

我整了个能"进化"的 Agent。原理是这样的：

1. 每次回答完，你按 **↑ 好** 或 **↓ 差**（不按也行，懒人友好）
2. 你退出的时候，它问你几个问题，大概就是"这次满意不"
3. 你用 `/export-dataset` 导出 DPO 数据集，拿去微调

**等等，DPO 是什么？**

Direct Preference Optimization，就是告诉模型"这个答案好，那个答案差"，然后它就朝着好的方向学。说白了就是**用你的嫌弃让它变聪明**。

> "你嫌我烂，我就想办法不烂，这就是进步" — ErnieCLI，大彻大悟

## v1.1.0 — MCP 支持，工具调用的工具调用

MCP（Model Context Protocol）就是**工具调用的扩展包**，
你可以接第三方 MCP server，让 Agent 有更多技能。

比如你接一个数据库 MCP，Agent 就能直接查你的表。
接一个 Figma MCP，Agent 就能读你的设计稿。

Baidu AI Studio 这边是后端代理的方案，你不用本地跑什么进程：

```yaml
mcp_servers:
  - type: sse
    url: https://你的mcp-server.com/sse
    server_label: my_tools
```

也支持运行时 `/mcp add` 热插拔，不需要重启。

> "接个 MCP 让 Agent 更聪明，然后发现还是不够聪明" — 某用户，2026
